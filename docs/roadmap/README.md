# Roadmap 문서 가이드

이 폴더는 현재 구현한 기능별로, 다음 버전에서 다룰 후속 Todo와 고도화 항목을 정리하는 공간이다.

## 기본 구조

- 버전 폴더를 상위에 두고 그 아래 도메인별 폴더 사용
- 예: `docs/roadmap/v2/documents/`, `docs/roadmap/v2/blocks/`
- 기능명 파일 사용
- 예: `block-delete.md`, `block-restore.md`

## 문서 목적

- 현재 구현한 기능의 기준선 정리
- 다음 버전에서 다룰 고도화 Todo 정리
- 추가 검증, 운영 고려사항, 정책 보강 포인트 기록
- 남아 있는 정책 질문 정리
- v1에서 기능 작동 우선으로 남겨둔 성능·운영 보강 과제를 추적

## 권장 섹션

- 현재 기준
- 후속 Todo
- 검증 보강
- 정책 / 운영 고려사항
- 선행조건 / 의존성
- 오픈 질문

## 작성 원칙

- 단순 아이디어 메모가 아니라 실제 후속 구현 시 참고할 수 있는 수준으로 적는다.
- 방금 구현한 기능 기준으로 남은 과제를 적는다. 아직 만들지 않은 전혀 다른 기능의 신규 API 목록을 적는 공간으로 쓰지 않는다.
- 같은 기능의 후속 항목은 흩어 쓰지 말고 해당 기능/버전 문서에 누적한다.
- 제품 요구사항으로 확정된 내용은 최종적으로 `docs/REQUIREMENTS.md`와 ADR에 반영한다.
