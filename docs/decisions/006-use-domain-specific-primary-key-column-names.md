# ADR 006: 엔티티 기본 키 컬럼명은 도메인별 `${domain}_id`를 사용한다

## 상태

채택됨

## 배경

현재 구현은 `Workspace`, `Document` 엔티티의 기본 키 프로퍼티는 모두 `id`이지만, 물리 DB 컬럼명도 동일하게 `id`를 사용하고 있었다.
이 방식은 테이블 간 조인, 운영 쿼리, 스키마 점검 시 어떤 도메인의 식별자인지 컬럼명만으로 식별하기 어렵다.
문서 서비스는 `workspace_id`, `document_id`, 향후 `block_id`처럼 도메인별 식별자가 자주 함께 등장하므로, 물리 컬럼명 수준에서 의미를 드러내는 규칙이 필요하다.

## 결정

- 엔티티의 자바 프로퍼티명은 기존처럼 `id`를 유지한다.
- 영속 스키마의 기본 키 컬럼명은 `${domain}_id` 형식을 사용한다.
- 현재 범위에서는 `workspaces.workspace_id`, `documents.document_id`를 적용한다.
- 향후 Block 구현 시 기본 키 컬럼명은 `blocks.block_id`를 사용한다.
- 외래 키 컬럼은 기존처럼 참조 대상을 드러내는 이름(`workspace_id`, `parent_id`, `document_id`)을 유지한다.

## 영향

- 장점:
  - 운영 쿼리와 스키마 점검에서 컬럼 의미가 더 명확해진다.
  - 다중 테이블 조인 시 `id` 충돌과 가독성 저하를 줄일 수 있다.
  - 향후 Block/추가 엔티티 도입 시 일관된 스키마 규칙을 적용할 수 있다.
- 단점:
  - 기존 스키마가 이미 배포된 환경이면 마이그레이션이 필요하다.
  - 자바 프로퍼티명과 물리 컬럼명이 달라져 매핑 규칙을 문서로 유지해야 한다.
