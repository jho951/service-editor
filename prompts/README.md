# 프롬프트 카탈로그

구현, 리뷰, 장애 대응에 재사용할 프롬프트를 정리합니다.

## 디렉토리 구조

- `prompts/*.md`
  - 날짜별 작업 로그

- `prompts/plans/`
  - 계획 템플릿과 중간 계획 문서

## 작업 로그 운영 원칙

- 작업 로그는 팀 또는 프로젝트 단위의 의미 있는 작업 기록이 필요할 때만 남깁니다.
- 개인 학습용 설명 요청, 간단한 질의응답, 잠깐 확인하고 끝나는 요청은 로그를 남기지 않습니다.
- 작업 로그는 가능한 한 목표 단위의 단일 파일로 유지합니다.
- 같은 목표 아래 이어지는 작업은 새 파일을 계속 만들지 말고 기존 파일에 `Step`을 추가하는 방식으로 누적합니다.
- 각 단계는 `## Step n. 제목` 형식으로 작성합니다.
- 각 Step 안에는 필요할 때 `목적`, `변경 내용`, `판단`, `검증`을 짧게 적어 흐름이 다시 읽히게 정리합니다.
- 새 파일은 목표가 달라졌거나, 기존 로그를 분리하는 편이 더 명확할 때만 만듭니다.
